# 🌟 OVERVIEW (ภาพรวมโปรเจกต์)
**Koo Mean SITE V.2** คือการอัปเกรดครั้งใหญ่ของศูนย์รวมลิงก์และพอร์ตโฟลิโอส่วนตัว เปลี่ยนจากเว็บ Static ธรรมดาให้กลายเป็นแพลตฟอร์มที่มีระบบหลังบ้านอัจฉริยะ สามารถควบคุมการมองเห็นเนื้อหาตามสิทธิ์ของผู้ใช้งาน (Role-Based Access Control) โดยยังคงความสวยงามในสไตล์ **Neon Dark Mode**

---

# ✨ KEY FEATURES (คุณสมบัติเด่น)

### 🔒 Role-Based Access Control (RBAC)
- ระบบคัดกรองลิงก์ตามสิทธิ์: **Public, VIP, Student และ Admin**
- ข้อมูลจะถูกดึงมาแสดงเฉพาะส่วนที่ผู้ใช้ได้รับอนุญาตเท่านั้น เพิ่มความปลอดภัยและความเป็นส่วนตัว

### 🔑 Secure Google Authentication
- ล็อกอินผ่าน **Google Sign-In** มาตรฐานระดับสากล
- ระบบจดจำเซสชันการใช้งาน ไม่ต้องล็อกอินซ้ำบ่อยๆ

### 🛠️ In-App Admin Dashboard
- จัดการระบบได้โดยไม่ต้องเปิด Google Sheets:
  - อนุมัติและเปลี่ยนสิทธิ์ผู้ใช้งาน (Roles)
  - แก้ไขหมวดหมู่และรายละเอียดลิงก์
  - ระบบปักหมุดลิงก์สำคัญ (**⭐ Pinned Links**)

### 🎨 Next-Gen UI/UX Design
- **Neon Glow Effect:** ดีไซน์เรืองแสงล้ำสมัยบนพื้นหลัง Dark Mode
- **Splash Screen:** ระบบโหลดหน้าเว็บแบบนุ่มนวล พร้อมแอนิเมชัน Fade-in
- **Real-time Search:** ค้นหาลิงก์และแอปที่ต้องการได้ทันที (Instant Filtering)

### 📊 System Health Monitor
- ตรวจสอบสถานะการเชื่อมต่อ API และสถานะของ Media ต่างๆ แบบ Real-time เพื่อความมั่นใจในการใช้งาน

---

# 🛠️ TECH STACK (เทคโนโลยีที่ใช้)

| ส่วนงาน | เทคโนโลยีที่ใช้ |
| :--- | :--- |
| **Frontend** | HTML5, CSS3 (Custom Animation), JavaScript (ES6+) |
| **Backend** | Google Apps Script (Web App API) |
| **Database** | Google Sheets (Real-time Database) |
| **Auth** | Google Identity Services (GSI) |
| **Icons** | FontAwesome 6.4.0 |

---

# 🚦 ROLE SYSTEM (ระบบระดับสมาชิก)

1. **🌍 Public:** ผู้เยี่ยมชมทั่วไป เห็นเฉพาะลิงก์สาธารณะและข้อมูลพื้นฐาน
2. **🌟 VIP / Student:** สิทธิ์พิเศษที่แอดมินปลดล็อคให้ จะเห็นลิงก์เฉพาะกลุ่ม (Private Links)
3. **⚡ Admin:** สิทธิ์สูงสุด สามารถเข้าถึงเมนู **Manage System** เพื่อควบคุมฐานข้อมูลทั้งหมด

---

# ⚙️ INSTALLATION (การตั้งค่า)
1. **Spreadsheet:** ตั้งชื่อแผ่นงาน `Users`, `Links`, `Profile` และ `Apps` ให้ถูกต้อง
2. **Web App:** Deploy โค้ด .gs และนำ `exec` URL มาใส่ใน `index.html`
3. **OAuth:** ตั้งค่า Client ID จาก Google Cloud Console
4. **Deploy:** ใช้งานผ่าน GitHub Pages หรือโฮสติ้งอื่นๆ ได้ทันที
